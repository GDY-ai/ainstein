- 所有数据库操作均通过 `database.py` 中的上下文管理器 `get_db()` 进行，确保事务安全与连接自动释放。
- LLM 交互统一使用 `agents/llm_client.py` 提供的 `call_llm` 或 `extract_json` 工具函数，实现提示词模板与解析逻辑的解耦。
- Agent 角色配置采用静态字典 `ROLES` 与数据库 `roles` 表同步的混合模式，通过 `RoleRegistry` 实现内存缓存与持久化的一致性。
- 研究引擎与 Agent 的思考产出均遵循严格的 JSON Schema 约束，并通过 `extract_json` 从 Markdown 代码块或混合文本中鲁棒地提取结构化数据。
