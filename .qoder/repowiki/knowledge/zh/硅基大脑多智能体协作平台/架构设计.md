- **事件驱动架构**：后端以 EventBus 为核心，将 Agent 行为、认知元素（CE）变更及用户操作解耦为异步事件；前端通过轮询 API 获取最新的知识图谱状态。
- **双轨数据模型**：系统同时维护传统的“项目-会话”线性研究流（Legacy）与基于图结构的“硅基大脑”非线性认知流（Brain），两者共享底层数据库但业务逻辑隔离。
- **编排与执行分离**：ATAOrchestrator 作为独立心跳线程监听事件并调度 Agent，确保主 Flask 进程不被长时 LLM 调用阻塞；前端通过 React Router 区分管理视图（BrainList）与可视化视图（BrainView）。
- **统一网关路由**：Flask 应用同时充当 API 服务器与静态资源服务器，所有前端请求经由 /ainstein/api/* 代理至后端逻辑，SPA 路由由 Flask 兜底返回 index.html。
